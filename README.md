# Showrun Media — Brand Operating System

> The substrate that makes every Showrun output look, sound, and feel like it came from the same expert team.

This repo is the canonical, machine-readable source for Showrun Media's brand. It compiles a single set of tokens, templates, and content fragments into every output we ship — capabilities deck, website, proposals, resumes, emails, social.

It's organized as a **three-pillar brand operating system**:

1. **Foundation** — design tokens (color, typography, spacing, radius, gradients, elevation, breakpoints) in W3C Design Tokens Community Group format
2. **Content** — canonical, reusable copy fragments (bios, methodology pitches, services, case studies, testimonials, FAQ) imported by every template
3. **Orchestration** — protocols and a sibling skill (`showrun-asset-generator`) that turns an intent ("make me a proposal") into a finished, branded artifact

## How to consume

### Web (Astro / Next.js / any CSS)
```bash
npm install @showrunmedia/tokens
```
```js
// In your global CSS
@import '@showrunmedia/tokens/css';

// Or for the dark theme overrides
@import '@showrunmedia/tokens/css/dark';

// Tailwind preset
// tailwind.config.js
const showrun = require('@showrunmedia/tokens/tailwind');
module.exports = { presets: [showrun], content: [...] };
```

### Code-generated documents (DOCX / PPTX / PDF)
```js
import tokens from '@showrunmedia/tokens';
// tokens.color.teal.deep.value === '#0096BC'
```

### n8n / agents / non-Node consumers
Fetch the flat JSON directly:
```
https://raw.githubusercontent.com/showrunmedia/showrun-design-system/main/build/json/tokens.json
```

### Claude Design (Anthropic Labs)
Onboard a new Claude Design project against this repo — Claude Design will read tokens, components, and content fragments to maintain brand consistency in any generated visual. Use for:
- Rapid visual prototyping (capabilities deck, brand portal previews)
- Fast-turnaround visuals (social posts, one-pagers, ad creative)
- Anywhere you want canvas-based iteration with the brand system pre-loaded

Export designs back into this repo via the per-template `output/` folders. Optionally export to Canva for collaborative edits.

### Canva, Gamma, Figma
Tokens are mirrored manually to each tool's brand kit until their MCP servers support brand-kit writes. See `docs/SYNCING_DESIGN_TOOLS.md` (v1.1).

## Version policy

- **v0.x — Candidate phase.** Tokens are marked `v0.1-candidate` and may change. The Phase 1 capabilities-deck prototype validates the system against real output. Phase 2 applies findings and tags `v1.0.0`.
- **v1.x — Stable.** Major.minor.patch. Token additions are minor bumps. Breaking changes (removed/renamed tokens) are major bumps. Released quarterly from the LEARNINGS log per `docs/OPERATING_CADENCE.md`.

## Three sources of truth

The brand operates on three sources of truth that work together:

| Source | Lives at | Authoritative for |
|---|---|---|
| **`showrun-brand-guidelines` skill** | `reftech12/de-ecosystem/SKILLS/showrun-brand-guidelines/` (private monorepo) | Strategy, voice, tone, positioning, design intent |
| **This repo** | `showrunmedia/showrun-design-system` (public) | Tokens, templates, content fragments, generated artifacts |
| **`showrun-asset-generator` skill** | `reftech12/de-ecosystem/SKILLS/showrun-asset-generator/` (private monorepo) | Orchestrating the flow: intent → template → content → output |

Each answers a different question. Together they're a complete brand OS.
