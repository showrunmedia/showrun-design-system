# content/

Canonical reusable copy fragments for Showrun Media. The "write once, use everywhere" library.

See [`content-conventions.md`](./content-conventions.md) for frontmatter spec, subdirectory ownership, and update rules.

## Subdirectories

- `identity/` — Daniel as a person (bios, narrative, headshot references)
- `positioning/` — How Showrun positions in market (one-liner, methodology, differentiation)
- `services/` — What Showrun sells (tier descriptions, pricing language, engagement escalation)
- `case-studies/` — Real or anonymized client engagement narratives
- `testimonials/` — Quotes, attribution, permission tracking
- `faq/` — Common-question answers in Showrun voice

## Status

v0.1 — empty scaffold. Entries get populated starting in Phase 1 (capabilities deck) as the deck demands them. Each subdirectory will get its own README when its first entry lands.
